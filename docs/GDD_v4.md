# 雷电战机 V4 - 游戏设计文档

## 1. 为什么2000年的雷电能做好，我们不行？

### 1.1 雷电成功的核心要素
- **精心设计的弹幕模式**：每种子弹都有明确的视觉特征（颜色、形状、速度）
- **节奏感**：敌人出现有固定节奏，弹幕有"呼吸感"
- **视觉层次**：背景、敌人、子弹、特效分层清晰
- **反馈即时**：击中、爆炸、拾取都有明显的视觉和音效反馈
- **难度曲线平滑**：从简单到地狱难度循序渐进

### 1.2 我们需要什么条件
- **美术资源**：专业的像素画师或AI生成高质量素材
- **音效系统**：射击、爆炸、拾取音效
- **弹幕设计**：精心设计的弹幕模式（环形、扇形、螺旋、随机等）
- **性能优化**：大量子弹时的流畅度
- **测试调优**：反复测试调整难度

## 2. 视觉表现升级方案

### 2.1 动态化贴图（不只是静态贴图）
- **玩家飞机**：
  - 引擎尾焰动画（多帧粒子）
  - 左右倾斜动画（移动时倾斜）
  - 射击时枪管发光
  - 受击时闪烁红屏

- **敌人**：
  - 入场动画（从屏幕外飞入）
  - 被击中时闪烁
  - 死亡时分解动画
  - 射击时武器发光

- **子弹**：
  - 旋转动画（螺旋弹）
  - 脉冲发光（呼吸效果）
  - 拖尾粒子（每帧生成）
  - 击中时爆开效果

### 2.2 特效清晰度提升
- **发光效果**：所有子弹添加外发光（shadowBlur）
- **颜色区分**：
  - 玩家子弹：青色/蓝色系
  - 敌人子弹：红色/橙色系
  - 道具：绿色/紫色系
- **大小对比**：子弹大小根据威力变化
- **拖尾长度**：根据子弹速度调整拖尾长度

## 3. 弹幕密度增强方案

### 3.1 敌人数量增加
- 基础敌人：每波15-30个（原8-25个）
- 敌人群体：3-10个一组（原3-8个）
- Boss弹幕：最多36发环形弹幕（原24发）
- 同时存在敌人上限：40个（原30个）

### 3.2 弹幕模式多样化
- **环形弹幕**：360度均匀分布
- **扇形弹幕**：60-180度扇形区域
- **螺旋弹幕**：旋转扩散
- **追踪弹幕**：追踪玩家位置
- **随机弹幕**：随机方向但固定速度
- **组合弹幕**：多种模式组合

### 3.3 难度曲线
- 第1-2波：简单，少量敌人
- 第3-5波：中等，开始有小群体
- 第6-9波：困难，大量弹幕
- 第10波+：地狱，密集弹幕+Boss

## 4. 音效系统（新增）
- 玩家射击音效
- 敌人射击音效
- 爆炸音效
- 拾取道具音效
- 擦弹音效
- 背景音乐

## 5. 性能优化
- 对象池：子弹、粒子、敌人复用
- 空间分割：碰撞检测优化
- 帧率控制：确保60fps
- 离屏剔除：不渲染屏幕外对象

## 6. 实现计划
1. 先实现动态化贴图和特效
2. 增加弹幕密度和模式
3. 添加音效系统
4. 性能优化
5. 测试调优
